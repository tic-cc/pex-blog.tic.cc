多角色、去中心的加密存证与权益合约

模块架构

  • 数字内容：任意的原生数字内容（例如文字、图片、视频等）或物质实体（手工作品、艺术品、建筑物等）的数字孪生内容
  • 去中心化内容寻址存储系统：例如 IPFS (The Interplanetary File System) 或 Arweave，能够去中心地、永久地、机密地存储数字内容，并通过独一无二、不可篡改的内容地址(CID)访问。也可降级为普通互联网，使用普通的 URL 作为内容地址。
  • 多角色加密存证与权益合约：简称加密存证，是将内容地址进行多角色公私钥加密，证明原始内容的存在性，并约定多角色权益合约的一段数据及代码。可类比于区块链的一个区块。
    • 多角色：围绕一种数字资产，通常具有多种不同的用户画像，例如铸造者、所有者、使用者、代理者、等等。
    • 加密存证：对内容地址，使用多种角色各自的公钥进行加密，只有该角色本人能够使用自己的私钥进行解密。
      • 可使用类似比特币的解锁脚本：<sig pubkey> <dup hash160 pubkeyhash equal checksig>（其中 sig 是对加密地址的签名） 来验证主人身份以及数字内容的存在性。
    • 权益合约：对不同的角色，权益合约定义了某角色的权益范围和条件，可以是价格、谜语、限额、代码等多种形式。例如，
      • 所有权合约：定义了所有权的获取、转让、期限、分成等约定，例如最简单的是一个价格。则有意愿获得所有权的人，必须支付给前任所有权人相应的金额，才能转让资产所有权给下一任所有权人。
      • 使用权合约：定义了使用权的获取、转让、期限等约定。例如最简单的是一个价格，则想要使用该资产的人，必须支付相应的费用，才能获得使用权，即可解密该存证，获取并使用该资产。
      • 代理权合约：定义了代理权的获取、转让、期限、分成等约定。接受用户委托，为铸造、加解密、使用、交易、合约验资等行为，提供随时随地的自动化服务，并按照约定获得相应的服务费，从而能够持续为用户提供更好的服务。
      • 包括但不限于上述角色，围绕资产还可有更多的细分或延展角色，例如经营权合约、处分权合约，等等。
  • 存证账本：记录加密存证的公开账本，可用专用的区块链或 NFT 智能合约实现。也可降级为普通数据库，甚至纸质账本。
  • 公证机：与外部世界对接，对权益合约进行公开验证的服务，可用区块链、零知识证明、预言机或其他去中心化验证技术来实现，也可降级为普通的银行、公证等传统中心信源。
  • 代理机：（由于普通用户不可能时刻在线响应）接受用户委托，为铸造、加解密、使用、交易、合约验资等行为，提供随时随地的自动化服务。例如作为一个去中心化应用 Dapp，或者降级为一个普通互联网的传统应用平台。

铸造流程

  1. 把数字内容存入去中心化内容寻址存储系统，获得一个内容地址。例如，在 IPFS 中以内容哈希作为地址，通过分布式哈希表 DHT、内容寻址(Content Addressing)等技术，使得地址和内容形成量子纠缠一样的双向一一映射效果。
  2. 对内容地址进行多角色公私钥加密，生成一段加密存证数据，数据包括
    • 对内容地址进行单向函数(one-way function)运算，获得地址索引。例如，进行再次哈希。
    • 铸造者本人，使用自己的公钥，对内容地址进行加密，得到铸造者加密地址。
    • 所有权人使用自己的公钥，对内容地址进行加密，得到所有者加密地址。
    • 代理机使用自己的公钥，对内容地址进行加密，得到代理机加密地址。
    • 为了允许所有权转让，绑定一份所有权合约。例如一份最简单的所有权合约：”向前任所有权人支付 1000 元”。
    • 为了允许该数字内容被其他用户所访问使用，绑定一份使用权合约。例如一份最简单的使用权合约：”向当前所有权人支付 1 元”。
    • 在这个技术框架内，可自由添加更多角色的权益合约。
    • 在这个技术框架内，可使用更复杂的加密算法，例如多人联合加密等。
  3. 将上述形成的存证数据文件，也存入去中心化内容寻址存储系统，获得一个存证地址。
  4. 把存证地址及其当前所有者记录到一个存证帐本中。通过公开的存证账本，可检索到任意存证并验证原始数字内容的存在。但是只有拥有相应权限的角色，才能够使用自己的私钥解开存证里的内容地址，最终访问到原始数字内容。

铸造者访问流程

  1. 铸造者通过代理机（例如一个去中心化应用 Dapp）浏览存证账本。
  2. 铸造者通过代理机，访问去中心化内容寻址系统，读取一个自己铸造的存证。
  3. 由于该存证中的铸造者加密地址是用该铸造者的公钥加密的，铸造者可以用自己的私钥，直接解密获得内容地址。
  4. 铸造者访问去中心化内容寻址系统，读取原始数字内容。

使用者访问流程

  1. 使用者通过代理机（例如一个去中心化应用 Dapp）浏览存证账本。
  2. 使用者通过代理机，访问去中心化内容寻址系统，读取一个自己感兴趣的存证。
  3. 使用者按照该存证中的使用权合约的要求，去其他环境中执行该合约。例如，该合约要求：”向当前所有权人的账号 xxx 支付 1 元人民币”，于是使用者向所有权人的指定账号支付 1 元人民币。
  4. 使用者通过代理机，要求验证自己的权益。代理机向公证机发出请求，公证机通过内建验证机制（区块链预言机等），确认该使用者已达成使用权合约。
  5. 代理机使用自己的私钥，解开代理机加密地址，从去中心化内容寻址存储系统中读取原始数字内容，返回给使用者。

所有权转让流程

  1. 用户通过代理机浏览存证账本。
  2. 用户通过代理机，访问去中心化内容寻址系统，读取一个自己感兴趣的存证。
  3. 用户按照该存证中的所有权合约的要求，去其他环境中执行该合约。例如，该合约要求：”向当前所有权人的地址 xxx 支付 1 比特币”，于是该用户向当前所有权人的指定账号支付 1 个比特币。
  4. 用户通过代理机向公证机发出请求，公证机通过内建验证机制（区块链预言机等），确认该用户已达成所有权合约。
  5. 代理机创建新的存证，把当前用户记录为新任所有权人，存入去中心化内容寻址系统，记入存证账本。
